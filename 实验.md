## 问题

### genisis 问题

**渲染问题无解**，训练可以 headless 训练，测试只能在本地有渲染功能的卡上跑

另外还有 pyglet 的**字体编码问题**：`pyglet.window.xlib.XlibException: Could not create UTF8 text property`

可以用这个简单的脚本进行测试，如果能正常使用，应该会看到一个框一闪而过

```py
import pyglet
# print(pyglet.window.xlib._have_utf8)
print("\nYES\n")
pyglet.window.Window()
```

建议分别在系统和 conda 环境中进行测试：

```py
python - <<'PY'
import os, locale, ctypes, ctypes.util
print({k: os.environ.get(k) for k in ("LANG", "LC_ALL", "LC_CTYPE", "XLOCALEDIR", "DISPLAY")})
print("python locale:", locale.setlocale(locale.LC_CTYPE, ""))

x11 = ctypes.CDLL(ctypes.util.find_library("X11") or "libX11.so.6")
x11.XSupportsLocale.restype = ctypes.c_int
x11.XSetLocaleModifiers.argtypes = [ctypes.c_char_p]
x11.XSetLocaleModifiers.restype = ctypes.c_char_p
print("XSupportsLocale:", x11.XSupportsLocale())
print("XSetLocaleModifiers:", x11.XSetLocaleModifiers(b"@im=none"))
PY
```

如果输出 `XSupportsLocale: 0`，说明当前进程里的 Xlib 不支持当前 LC_CTYPE locale

- 系统有问题：

```sh
sudo apt-get update
sudo apt-get install -y locales libx11-6 libx11-data
sudo locale-gen en_US.UTF-8
sudo update-locale LANG=en_US.UTF-8
unset XLOCALEDIR
```

- 系统没问题但 conda 环境有问题：`export XLOCALEDIR=/usr/share/X11/locale`，将 XLOCALEDIR 指向系统 X11 locale 数据即可
